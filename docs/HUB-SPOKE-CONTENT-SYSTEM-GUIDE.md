## Part 1: Understanding the Hub + Spoke Model

### What Is Hub + Spoke Content?

**Hub Page:** A comprehensive guide (3,000-5,000 words) that covers a main topic broadly, with internal links to detailed supporting content. The hub serves as the central resource and primary ranking target.

**Spoke Pages:** Individual blog posts (1,500-2,200 words each) that cover specific subtopics in depth. Each spoke links back to the hub and cross-links to related spokes.

### Why Hub + Spoke Works

The model creates topical authority by:
- Demonstrating comprehensive expertise on a subject
- Creating internal linking structures that signal relevance
- Capturing traffic across multiple search intents
- Guiding users through complete decision journeys
- Providing AI systems with clear content relationships

### Key Success Factors

1. **Strategic Planning:** Thorough pre-work before content creation
2. **Intent Alignment:** Each piece serves a specific user intent
3. **Verified Information:** All content based on project files only
4. **Consistent Voice:** Brand voice maintained throughout
5. **Technical Precision:** Proper SEO, schema, and linking
6. **Quality Control:** Systematic review and validation

#### Spoke Topic Identification

Each spoke must be categorized by customer journey stage:

**Awareness Stage Spokes (Educational)**
- Focus: Problem identification, symptom recognition, general education
- User intent: Understanding what's happening and why it matters
- Content depth: Broad overview, foundational concepts

**Consideration Stage Spokes (Comparative)**
- Focus: Solution options, process explanations, comparisons
- User intent: Evaluating available solutions and tradeoffs
- Content depth: Detailed analysis, pros/cons, decision factors

**Decision Stage Spokes (Commercial)**
- Focus: Service details, trust signals, conversion facilitation
- User intent: Choosing a provider and taking action
- Content depth: Specific offerings, local focus, clear CTAs

## Appendix: Common Pitfalls and Solutions

### Pitfall 1: Intent Leakage

**Problem:** Sections contain content that serves a different intent than the page's primary purpose.

**Solution:**
- Complete intent brief before writing
- Review each section against declared intent
- Move off-topic content to appropriate page
- Link to other pages instead of covering tangentially

### Pitfall 2: Generic Content

**Problem:** Content lacks specific local knowledge and client expertise.

**Solution:**
- Require verified local information in project files
- Include specific location references (not just city name insertion)
- Reference genuine local conditions, codes, challenges
- Stop and request information rather than write generically

### Pitfall 3: Weak Internal Linking

**Problem:** Content pieces exist in isolation without strategic connections.

**Solution:**
- Complete linking map before writing
- Place links at intent transition points
- Use descriptive anchor text
- Verify all links fulfill their signaled intent

### Pitfall 4: Failed Summarizability

**Problem:** Sections don't stand alone; page purpose unclear from summaries.

**Solution:**
- Apply summarizability test during validation
- Restructure sections that can't be summarized
- Ensure each section has distinct, clear purpose
- Remove redundant or overlapping content

### Pitfall 5: Missing Verification

**Problem:** Content contains assumed or fabricated information.

**Solution:**
- Stop content creation when information is missing
- Use standard information request format
- Never proceed with unverified claims
- Mark uncertain sections with [NEEDS VERIFICATION]
